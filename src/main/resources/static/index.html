What is JWT?

JSON Web Token (JWT) is an open standard (RFC 7519) that defines a compact and self-contained way for securely transmitting information between parties as a JSON object. This information can be verified and trusted because it is digitally signed.

Spring Boot

Spring Boot is an open-source Java framework that was developed by Pivotal in 2014 and simplifies the task of deploying Java enterprise web applications. It is a project built on top of the Spring framework that provides an efficient way to set up and run applications.

Features

Expose endpoints for login(with remember Me), Sign Up. Support for Forgot Password, Validate Account using E-mail Service. GET/PUT/DELETE mapping for managing Account of the logged-in user. Restrict endpoints based on user ROLE i.e ADMIN, MODERATOR, USER.

Access to all Users
  • email/api/auth/signup
    If roles are not provided,Default role is ROLE_USER.

    Roles can be provided as shown in below example
  • email/api/auth/activate
    This endpoint can regenerate verification email if previous link expires.
  • get_app/api/auth/activate
    A GET method embedded in activation link to activate account.
    Just click on the activate button sent to your email.
  • email/api/auth/login
    POST Method to log in and get JWT token if credentials are valid.

    In case of invalid credentials.
    If Account is not activated after Sign up.
  • email/api/auth/reset-password
    POST Method to reset password. The OTP is sent in user email.

    In case of invalid OTP.
  • get_app/api/auth/reset-password
    GET method to send reset password request. OTP is sent to email.

Access to Logged In user (user with jwt token in authentication header)
  • get_app/api/user
  • input/api/user
  • delete/api/user

ROLE based protected endpoints for Logged In users
  • get_app/api/person
    Access to user having role ROLE_USER
  • get_app/api/moderator
    Access to user having role ROLE_MODERATOR
  • get_app/api/hiddenmessage
    Access to user having role ROLE_ADMIN
